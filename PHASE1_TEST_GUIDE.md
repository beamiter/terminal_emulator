# Phase 1 功能测试指南

本指南列出了 Phase 1 所有功能的测试步骤和预期行为。

## 前置条件

1. 编译项目：
```bash
cargo build --release
```

2. 运行应用：
```bash
./target/release/terminal_emulator
```

---

## Phase 1.1: 搜索功能

### 测试 1.1.1: 打开/关闭搜索面板
**步骤**：
1. 在终端中运行一些命令生成输出（如 `ls -la`）
2. 按 `Ctrl+Shift+F` 打开搜索面板
3. 搜索面板应该在右上角出现

**预期**：
- ✓ 搜索面板在右上角（距离右边约 10px）
- ✓ 搜索输入框获得焦点
- ✓ 面板显示搜索框、按钮和计数

**可能的问题**：
- [ ] 搜索面板不出现
- [ ] 搜索框没有焦点
- [ ] 面板位置不对

---

### 测试 1.1.2: 实时搜索
**步骤**：
1. 打开搜索面板 (Ctrl+Shift+F)
2. 在搜索框中输入 "ls"
3. 观察搜索结果

**预期**：
- ✓ 搜索结果实时显示（不需要按 Enter）
- ✓ 匹配项被高亮（反色显示）
- ✓ 显示匹配计数（如 "3/5"）
- ✓ 搜索框中有文本时不能切换到其他快捷键

**可能的问题**：
- [ ] 搜索不工作
- [ ] 高亮不显示
- [ ] 计数错误
- [ ] 可以在搜索中执行其他快捷键

---

### 测试 1.1.3: 导航匹配项
**步骤**：
1. 搜索一个常见词（如 "root" 或 ".."）
2. 按 `Enter` 导航到下一个匹配
3. 按 `Shift+Enter` 导航到上一个匹配

**预期**：
- ✓ Enter 跳转到下一个匹配项，显示 "2/5" → "3/5" 等
- ✓ Shift+Enter 跳转到上一个匹配项
- ✓ 终端自动滚动显示当前匹配项
- ✓ 当前匹配项高亮更深

**可能的问题**：
- [ ] 导航不工作
- [ ] 没有自动滚动
- [ ] 当前匹配项高亮不明显

---

### 测试 1.1.4: 正则表达式搜索
**步骤**：
1. 打开搜索面板
2. 在搜索框左侧看 `.*` 按钮（应该是灰色）
3. 在搜索框中输入一个正则表达式（如 `root|user` 或 `\d+`）
4. 观察搜索结果

**预期**：
- ✓ 搜索结果显示匹配正则表达式的项
- ✓ 如果正则表达式无效，显示红色错误提示
- ✓ 可以按 `.*` 按钮切换正则表达式模式

**可能的问题**：
- [ ] 正则表达式不工作
- [ ] 错误提示不显示
- [ ] 按钮无法点击

---

### 测试 1.1.5: 关闭搜索
**步骤**：
1. 打开搜索面板
2. 按 `Escape`
3. 按 `Ctrl+Shift+F` 再次打开
4. 观察搜索框是否清空

**预期**：
- ✓ Escape 关闭搜索面板
- ✓ 高亮仍然保留（可选，取决于配置）
- ✓ 再次打开时搜索框清空

**可能的问题**：
- [ ] Escape 不关闭搜索
- [ ] 搜索框没有清空

## Phase 1.3: 快捷键可配置化

### 测试 1.3.1: 默认快捷键
**步骤**：
1. 运行应用
2. 测试默认快捷键（参考 KEYBINDINGS.md）
3. 尝试这些快捷键：
   - `Ctrl+Shift+T` - 新建会话
   - `Ctrl+W` - 关闭会话
   - `Ctrl+Tab` - 下一个会话
   - `Ctrl+Shift+Tab` - 上一个会话

**预期**：
- ✓ 所有默认快捷键工作
- ✓ 会话正确创建/关闭/切换

**可能的问题**：
- [ ] 快捷键不工作
- [ ] 会话管理有问题

---

### 测试 1.3.2: 配置文件加载
**步骤**：
1. 创建配置文件：
```bash
mkdir -p ~/.config/terminal_emulator
cat > ~/.config/terminal_emulator/keybindings.toml << 'EOF'
[keybindings]
"alt+n" = "session:new"
"alt+w" = "session:close"
"alt+j" = "session:next"
"alt+k" = "session:prev"
"ctrl+shift+c" = "edit:copy"
"ctrl+shift+v" = "edit:paste"
"ctrl+shift+f" = "search:open"
EOF
```

2. 重启应用
3. 尝试新的快捷键（`Alt+N` 创建会话）

**预期**：
- ✓ 配置文件被正确加载
- ✓ 自定义快捷键工作
- ✓ 默认快捷键不再工作

**可能的问题**：
- [ ] 配置文件不加载
- [ ] 快捷键不生效
- [ ] 加载时出错（应该显示默认快捷键）

## 调试提示

如果出现问题：

1. **查看日志**：检查终端输出中是否有错误信息
2. **重启应用**：某些状态问题可以通过重启解决
3. **重置配置**：删除 `~/.config/terminal_emulator/keybindings.toml`，使用默认快捷键再试
4. **检查权限**：对于链接打开，确保 `xdg-open`（Linux）或 `open`（macOS）可用
5. **字体问题**：如果搜索/命令调色板 UI 看起来很奇怪，可能是字体问题
